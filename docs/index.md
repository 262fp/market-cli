# FUNDAMENTOS DE PROGRAMACIÓN 2026 (Versión Final)

## Producto del curso

**Aplicación de consola para la gestión de una entidad de negocio (Producto, Cliente, Usuario u otra equivalente), desarrollada mediante programación estructurada, integrando estructuras secuenciales, condicionales, repetitivas, subprogramas, recursividad, estructuras de datos básicas y archivos para registrar, consultar, actualizar, eliminar, procesar y presentar información.**

---

# UNIDAD 1 – Programación secuencial y condicional

### Resultado de aprendizaje

Aplica estructuras secuenciales y condicionales para resolver problemas simples y construir un menú básico para la gestión de una entidad.

### Producto de la unidad

**Menú básico de gestión de una entidad.**

| Sesión | Contenido | Actividad práctica | Producto de la sesión |
|---------|-----------|--------------------|-----------------------|
| **S1** | Algoritmos, resolución de problemas, pseudocódigo y entorno de desarrollo | Resolver problemas algorítmicos y crear el proyecto base | Proyecto de consola creado con el menú principal y estructura inicial. |
| **S2** | Variables, tipos de datos, operadores y entrada/salida | Resolver problemas secuenciales e implementar el registro de una entidad | Opción **Registrar** implementada. |
| **S3** | Condicionales simples y compuestos (`if`, `if-else`) | Resolver problemas condicionales y aplicarlos al proyecto | Validaciones y búsqueda básica implementadas. |
| **S4** | Condicionales múltiples (`switch`) y menús | Construir el menú principal del proyecto | Menú básico completamente funcional. |
| **S5** | Evaluación Unidad 1 | Exposición del producto | **Producto U1:** Menú básico de gestión de una entidad. |

---

# UNIDAD 2 – Procesamiento de datos y programación modular

> **U2 = U1 + nuevas capacidades**

### Resultado de aprendizaje

Implementa estructuras repetitivas, subprogramas, recursividad y estructuras de datos básicas para desarrollar una aplicación de consola con operaciones CRUD en memoria y procesamiento de información.

### Producto de la unidad

**Aplicación de consola con CRUD en memoria y procesamiento de datos.**

| Sesión | Contenido | Actividad práctica | Producto de la sesión |
|---------|-----------|--------------------|-----------------------|
| **S6** | Estructura repetitiva **`for`** y **arreglos** | Implementar el almacenamiento y recorrido de registros utilizando arreglos. | CRUD básico utilizando arreglos (Registrar y Listar). |
| **S7** | Estructuras repetitivas **`while`** y **`do-while`** | Implementar menús interactivos, validación de entradas y control del flujo. | Menú interactivo con validaciones. |
| **S8** | **Algoritmos de búsqueda y ordenación** | Implementar búsqueda lineal, búsqueda binaria (conceptual) y algoritmos básicos de ordenación aplicados al CRUD. | CRUD con búsqueda y ordenación de registros. |
| **S9** | **Subprogramas (funciones), parámetros, retorno y recursividad** | Modularizar el proyecto e implementar algoritmos recursivos clásicos. | CRUD modular y laboratorio de algoritmos recursivos. |
| **S10** | **Matrices y diccionarios (HashMap) como estructuras de datos para problemas específicos** | Resolver problemas utilizando matrices y estructuras clave–valor como alternativa según el tipo de problema. | Procesamiento de datos utilizando matrices y diccionarios. |
| **S11** | **Consultas, agregación y procesamiento de datos** | Implementar consultas, filtros, agregaciones (**COUNT, SUM, AVG, MIN, MAX**), agrupaciones básicas, estadísticas descriptivas y reportes sobre la información almacenada. | Aplicación con consultas, agregaciones y procesamiento de datos. |
| **S12** | Evaluación Unidad 2 | Exposición del producto. | **Producto U2:** Aplicación de consola con CRUD en memoria y procesamiento de datos. |

---

# UNIDAD 3 – Persistencia e integración del producto

> **U3 = U1 + U2 + integración final**

### Resultado de aprendizaje

Integra persistencia de datos y generación de salidas en la aplicación desarrollada durante el curso.

### Producto de la unidad

**Aplicación de consola completa para la gestión de una entidad.**

| Sesión | Contenido | Actividad práctica | Producto de la sesión |
|---------|-----------|--------------------|-----------------------|
| **S13** | Archivos (lectura y escritura) | Implementar persistencia mediante almacenamiento y recuperación de la información. | Aplicación con persistencia de datos. |
| **S14** | Generación de reportes (Excel, PDF e impresión) | Exportar información e imprimir reportes o comprobantes utilizando los datos del sistema. | Reportes generados en Excel, PDF e impresión de información. |
| **S15** | Sustentación del producto del curso | Presentación técnica de la aplicación desarrollada durante el semestre. | **Producto Final:** Aplicación de consola completa sustentada. |
| **S16** | Evaluación final | Evaluación teórico-práctica. | Examen final. |
